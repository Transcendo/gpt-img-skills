# GPT Image Skills

这个仓库用于沉淀可复用的生图风格 skill，面向自媒体内容生产、短视频封面、图文账号配图和系列化视觉栏目。

目标不是收集零散 prompt，而是把稳定的视觉风格、平台构图规则、默认行为和失败修正方法整理成可直接调用的 skill。每个 skill 都应该让使用者用一句主题就能得到风格一致、适合发布的图像结果。

## 仓库结构

```text
skills/
  megastructure-img/
    SKILL.md
    agents/
      openai.yaml
    references/
      prompt-patterns.md
```

约定：

- 每个风格一个独立目录：`skills/<skill-name>/`。
- `SKILL.md` 是主入口，写清触发场景、默认行为、内部构图规则和输出约束。
- `agents/` 放面向具体 agent 或平台的展示配置。
- `references/` 放可选的提示词模式、失败修正、风格扩展资料。
- 默认优先产出图像，不把复杂 prompt 展示给最终用户，除非用户明确要求。

## 使用方式

把需要的 skill 目录复制到本地 agent 的 skills 目录后，即可按 skill 名调用。例如：

```text
使用 $megastructure-img 生成：抖音竖屏，末日雪原尽头的巨型城市墙
```

`megastructure-img` 会自动根据“横屏 / 竖屏 / B站 / 抖音 / 小红书”等词判断画幅，并生成适合社交媒体封面使用的巨构图。

## 新增 Skill 标准

新增一个生图风格时，建议至少包含：

1. 明确的风格定位：它解决哪类自媒体视觉需求。
2. 触发词：用户说什么时应该自动启用。
3. 默认行为：是否直接生图、是否解释 prompt、是否需要追问。
4. 平台构图：横屏、竖屏、封面、安全区、文字避让等规则。
5. 视觉核心：主体、构图、光线、材质、情绪、色彩、禁忌项。
6. 失败修正：常见跑偏结果如何拉回风格。
7. 示例主题：便于快速 smoke test。

## 内容方向

后续可以继续沉淀：

- 赛博城市、废土公路、末日建筑等系列风格。
- 国风神话、山海经、东方奇观等文化风格。
- 商业海报、产品氛围图、账号封面等运营风格。
- 横屏视频封面、竖屏短视频封面、小红书首图等平台专用风格。

这个 repo 的核心价值是让可复用的生图能力逐步产品化：少写一次性 prompt，多沉淀可调用、可迭代、可发布的风格资产。
