# GPT Image Skills

这个仓库用于沉淀可复用的生图风格 skill，面向自媒体内容生产、短视频封面、图文账号配图和系列化视觉栏目。

目标不是收集零散 prompt，而是把稳定的视觉风格、平台构图规则、默认行为和失败修正方法整理成可直接调用的 skill。每个 skill 都应该让使用者用一句主题就能得到风格一致、适合发布的图像结果。

## 当前收录

| Skill | 方向 | 适合场景 |
| --- | --- | --- |
| [`megastructure-img`](skills/megastructure-img/SKILL.md) | 静态巨构崇高视觉系统：末日废土、奇妙梦境、外星/泰坦文明等 route 化主题转译 | B站横屏封面、抖音/小红书竖屏封面、科幻氛围图、巨物叙事视觉、系列化图像栏目 |

## 仓库结构

```text
skills/
  megastructure-img/
    SKILL.md
    agents/
      openai.yaml
    references/
      prompt-patterns.md
      route-smoke-tests.md
```

约定：

- 每个风格一个独立目录：`skills/<skill-name>/`。
- `SKILL.md` 是主入口，写清触发场景、默认行为、内部构图规则和输出约束。
- `agents/` 放面向具体 agent 或平台的展示配置。
- `references/` 放可选的提示词模式、失败修正、风格扩展资料。
- 默认优先产出图像，不把复杂 prompt 展示给最终用户，除非用户明确要求。

## 使用方式

把需要的 skill 目录复制到本地 agent 的 skills 目录后，即可按 skill 名调用。例如：

```text
使用 $megastructure-img 生成：抖音竖屏，末日雪原尽头的巨型城市墙
```

`megastructure-img` 会自动根据“横屏 / 竖屏 / B站 / 抖音 / 小红书”等词判断画幅，并根据主题选择内部 route，把主题转译成适合社交媒体封面使用的静态巨构图。默认优先静态图，不主动把画面视频化；巨构图的震撼来自凝视、静止、不可测量和不可解释。

### `megastructure-img` 当前核心 route

| Route | 中文方向 | 心理机制 | 典型主题 |
| --- | --- | --- | --- |
| Wasteland Relic | 末日废土 / 旧世界遗迹 | 旧世界已死，但基础设施仍以过大的尺度压住现在 | 干涸海床里的水坝、废弃太空港、断裂高速、旧机场、荒原城市墙 |
| Dream Megastructure | 奇妙梦境巨构 | 熟悉空间的逻辑温柔失效，醒来后仍然忘不掉 | 云上街道、水下教室、天空楼梯、漂浮住宅、黄昏 playground 后的巨墙 |
| Titan Alien Relic | 外星 / 泰坦文明遗迹 | 人类规则完全不适用，人类只是迟到的微小见证者 | 冰卫星信号塔、恒星环残片、非欧几何方碑群、星球表面的巨门 |

这些 route 仍属于同一个主 skill，不拆成多个 skill。只有当某条 route 脱离“巨构崇高”也能稳定服务独立需求时，才考虑拆分。

如果需要发 B站、抖音或小红书，可以要求输出发布辅助信息：

```text
使用 $megastructure-img 生成一组 B站横屏图：雪原尽头的巨型城市墙，并给标题、简介和标签
```

约定：

- 默认图片本体保持无字、无 logo、无水印，标题和简介作为外部发布文案输出。
- `一组图 / 组图 / 系列图 / 一套图` 默认表示围绕一个主题产出多张视觉变体，而不是多个互不相关的主题。
- 发布辅助信息包括：组图主题、主标题、每张图的小标题、简介、标签、备用标题。

## Roadmap：后续 route

`megastructure-img` 后续优先沉淀 7 条路线。它们先作为 roadmap，不急着塞进默认生成逻辑，避免主 skill 过早膨胀。

| Route | 中文方向 | 核心机制 | 避坑 |
| --- | --- | --- | --- |
| Concretecore Liminal | 砼核阈限 | 熟悉公共空间失去正常功能：混凝土大厅、停车场、地铁站、学校走廊变得无人、无标识、尺度不对 | 不做鬼屋、不做普通建筑摄影、不复刻 backrooms 黄墙 |
| Deep Time Geology | 深时地质文明 | 文明不是毁灭，而是沉积成地层、化石和山体切面 | 不做普通废墟，不把画面堆满碎石 |
| Submerged / Frozen Civilization | 沉没 / 冰封文明 | 世界被水、冰、雾等介质慢慢吞掉，只剩结构残影 | 不做深海恐怖、不做亚特兰蒂斯宫殿 |
| Industrial Cathedral | 工业圣殿 | 基础设施大到像宗教：水坝、核电站、数据中心、船坞、发射井 | 不做普通工业摄影、不做赛博朋克工厂 |
| Solar Relic | 神性光构 | 光不是装饰，而是无法靠近的结构压力 | 不做金碧辉煌、不做奇幻宫殿、不做治愈圣光 |
| Post-Human Ecology | 后人类生态 | 自然继续生长，人类不再重要：鹿群、麦田、森林、鸟群成为时间尺 | 不做环保治愈、不做可爱动物主角 |
| Cold War Retro-Future | 冷战遗迹 / 旧未来主义 | 一个曾经相信未来的时代留下过大的壳：雷达阵列、太空港、地下指挥大厅、月球基地 | 不做怀旧海报、不做蒸汽朋克 cosplay、不放标语 logo |

这些 route 的判断标准：先看能否服务“微小见证者 + 压倒性主形 + 熟悉世界异常化 + 静态电影感”。如果不能，就不进入主 skill。

## 新增 Skill 标准

新增一个生图风格时，建议至少包含：

1. 明确的风格定位：它解决哪类自媒体视觉需求。
2. 触发词：用户说什么时应该自动启用。
3. 默认行为：是否直接生图、是否解释 prompt、是否需要追问。
4. 平台构图：横屏、竖屏、封面、安全区、文字避让等规则。
5. 视觉核心：主体、构图、光线、材质、情绪、色彩、禁忌项。
6. 失败修正：常见跑偏结果如何拉回风格。
7. 示例主题：便于快速 smoke test。

## 内容方向

近期重点不是新增一堆独立 skill，而是把 `megastructure-img` 做成主 mega skill：一个稳定基调，多条 route，默认静态图优先。当前已经落定 Wasteland Relic、Dream Megastructure、Titan Alien Relic 三条核心路线；roadmap 中的 7 条路线会按稳定性逐步进入主 skill。

这个 repo 的核心价值是让可复用的生图能力逐步产品化：少写一次性 prompt，多沉淀可调用、可迭代、可发布的风格资产。
